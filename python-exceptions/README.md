# Python - Exceptions

This project covers exception handling in Python — understanding the difference between errors and exceptions, using `try`/`except`/`finally` blocks, and raising built-in exceptions.

## Learning Objectives

- Difference between errors and exceptions
- How to handle exceptions with `try` / `except`
- How to raise built-in exceptions
- When and why to use a `finally` clause for clean-up actions

## Requirements

- All files interpreted on Ubuntu 20.04 LTS using Python 3.8.5
- Code follows `pycodestyle` (version 2.7.*)
- All files must be executable and end with a new line
- First line of all files: `#!/usr/bin/python3`

## Tasks

| # | File | Description |
|---|------|-------------|
| 0 | `0-safe_print_list.py` | Print `x` elements of a list using `try`/`except` (no `len()`) |
| 1 | `1-safe_print_integer.py` | Print an integer with `"{:d}".format()`, return `True`/`False` (no `type()`) |
| 2 | `2-safe_print_list_integers.py` | Print only integers from the first `x` elements of a list |
| 3 | `3-safe_print_division.py` | Divide two integers, print result in `finally`, return result or `None` |
| 4 | `4-list_division.py` | Element-wise division of two lists with specific error messages |
| 5 | `5-raise_exception.py` | Raise a `TypeError` |
| 6 | `6-raise_exception_msg.py` | Raise a `NameError` with a custom message |
